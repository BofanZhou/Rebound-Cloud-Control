# 回弹云控管理系统 —— 使用说明书

## 1. 系统概述

回弹云控管理系统是一款面向钢管折弯加工场景的 Web 端控制平台。系统集成了设备监控、回弹参数智能推荐、历史记录追溯、AI 模型训练、多机管理与用户权限控制等功能，旨在帮助操作员快速获取最优折弯补偿参数，提升加工精度与效率。

## 3. 安装与部署

### 3.1 获取源代码

```bash
git clone https://github.com/BofanZhou/Rebound-Cloud-Control.git
cd Rebound-Cloud-Control
```

### 3.2 安装后端环境

```bash
cd backend

# 建议创建虚拟环境
python -m venv venv
.\venv\Scripts\activate        # Windows
# source venv/bin/activate     # Linux/macOS

# 安装依赖
pip install -r requirements.txt

cd ..
```

### 3.3 安装前端环境

```bash
cd frontend
npm install
cd ..
```

### 3.4 启动系统

#### 一键启动（推荐）

```bash
# 方式 1：Python 脚本（跨平台）
python start.py

# 方式 2：PowerShell 脚本（Windows 推荐）
.\start.ps1

# 方式 3：批处理（Windows）
start.bat
```

启动成功后：
- 前端地址：`http://localhost:5173`
- 后端地址：`http://localhost:8000`
- API 文档：`http://localhost:8000/docs`

#### 手动启动

如需分别调试前后端，可开两个终端：

**终端 A（后端）：**
```bash
cd backend
python main.py
```

**终端 B（前端）：**
```bash
cd frontend
npm run dev
```

## 4. 系统功能说明

### 4.1 登录模块

系统支持两种登录方式：

#### 机器登录
- 适用于单设备直连场景。
- 在登录页选择"机器登录"，输入机器编号（如 `MCH-001`）。
- 若机器不存在，系统会自动创建并进入该设备控制界面。

#### 用户登录
- 适用于多机管理场景。
- 在登录页选择"用户登录"，输入用户名和密码。
- 登录后进入机器选择页，可选择已绑定的设备进行操控。

### 4.2 设备监控（Device）

- 实时展示设备运行状态（运行中/待机/故障/离线）。
- 显示关键工艺参数：温度、压力、当前角度、补偿量等。
- 支持手动提交折弯任务，查看任务执行进度。
- 设备状态通过 WebSocket 实时推送更新。

### 4.3 参数推荐（Recommend）

- 输入钢管材质、管径、壁厚、目标角度等参数。
- 系统基于内置规则引擎计算回弹补偿值，给出推荐工艺参数。
- 支持将推荐结果一键下发至设备执行。

### 4.4 历史记录（History）

- 查看所有折弯作业的历史数据。
- 支持按机器编号、时间范围、材质等条件筛选。
- 可查看单次作业的详细参数与结果对比。

### 4.5 模型训练（Training）

- **数据集上传**：上传 CSV 格式的历史加工数据（需包含材质、管径、壁厚、角度、回弹量等字段）。
- **在线训练**：配置训练参数（Epoch、Batch Size、学习率），启动 PyTorch 神经网络模型训练。
- **训练监控**：实时查看 Loss 曲线与验证准确率。
- **迭代预测**：输入目标参数，系统自动迭代逼近最优补偿角。

### 4.6 多机管理（Machine Management）

- 查看当前系统中注册的所有生产设备列表。
- 添加新机器：输入机器编号、名称、位置等信息。
- 编辑或删除已有机器信息。
- 切换当前操控的目标机器。

### 4.7 用户管理（User Management）

> 仅管理员角色可见。

- 查看系统内所有注册用户。
- 创建新用户：设置用户名、姓名、密码与角色（管理员/维修员/操作员）。
- 删除用户或重置用户密码。

### 4.8 操作日志（Operation Log）

> 仅管理员角色可见。

- 记录用户的登录、参数修改、任务提交等关键操作。
- 支持按用户名、操作类型、时间范围筛选。
- 用于安全审计与问题追溯。

## 6. 常见问题

### Q1：启动后前端无法连接后端？
- 确认后端服务已正常启动（访问 `http://localhost:8000/health` 应返回 `healthy`）。
- 确认前端 `vite.config.ts` 中的代理配置正确（默认已配置 `/api` 转发到 `localhost:8000`）。

### Q2：模型训练报错或速度很慢？
- PyTorch 默认安装 CPU 版本，训练速度取决于 CPU 性能。
- 若需 GPU 加速，请根据 NVIDIA 驱动版本安装对应 CUDA 版本的 PyTorch。

### Q3：如何修改服务端口号？
- 后端端口：修改 `backend/main.py` 中 `uvicorn.run` 的 `port` 参数。
- 前端端口：修改 `frontend/vite.config.ts` 中 `server.port`。

### Q4：数据库文件在哪里？
- 默认 SQLite 数据库位于 `backend/data/rebound.db`。
- 如需更换数据库，可设置环境变量 `DATABASE_URL`（如 PostgreSQL 连接串）。

### Q5：如何停止服务？
- 一键启动时按 `Ctrl + C` 即可同时终止前后端进程。
- 或运行 `stop.bat` / `stop.ps1`。
