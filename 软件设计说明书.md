# 回弹云控管理系统 V2.0 软件设计说明书

---

## 一、软件概述

### 1.1 软件全称与版本
- **软件全称**：回弹云控管理系统 V2.0
- **软件简称**：Rebound Cloud Control（ReboundCC）
- **版本号**：V2.0

### 1.2 开发背景与目标

在钢管弯曲加工过程中，金属材料存在弹性回复现象（即"回弹"），导致实际弯曲角度与目标角度存在偏差。传统生产方式依赖操作工人的经验进行补偿，精度不稳定、效率低下、质量难以追溯。

**本系统的核心目标**是：
- 通过云端 AI 模型精准预测钢管回弹补偿值
- 实现补偿参数的自动计算、分发与下发
- 提供多设备的实时监控与统一管理
- 建立完整的加工质量追溯体系

### 1.3 技术领域
- 工业物联网（IIoT）
- 智能制造
- 边缘计算
- 物理信息神经网络（Physics-Informed Neural Network, PINN）

### 1.4 部署方式
- **架构**：B/S 架构（浏览器/服务器）
- **部署**：云端部署 + 边缘节点
- **访问**：支持 PC、平板、手机跨平台访问
- **网络**：支持局域网内网部署，也支持公网 SaaS 模式

## 三、用户角色与权限体系

### 3.1 角色定义

| 角色 | 权限范围 | 典型用户 |
|---|---|---|
| **系统超级管理员** | 所有功能，包括用户管理、系统配置、OTA 升级 | 企业 IT 管理员 |
| **区域管理员** | 管理辖区内所有机器、查看所有数据、生成报表 | 车间主任、区域经理 |
| **维修人员** | 查看设备状态、历史记录、接收告警通知 | 设备维护工程师 |
| **操作员** | 操作指定机器、提交加工任务、查看本机历史 | 现场操作工人 |

### 3.2 认证方式

1. **用户登录**：用户名 + 密码 + 角色权限
2. **机器登录**：机器 ID 直接登录（适用于现场快速操作）
3. **双因素认证**（可选）：短信/邮箱验证码
4. **Token 机制**：JWT + 刷新令牌，支持单点登录和主动失效

## 四、功能模块详细设计

### 4.1 智能参数推荐模块

**功能描述**：根据输入的钢管参数，计算并推荐最佳回弹补偿值。

**输入参数**：
- 管径（10~500 mm）
- 壁厚（1~50 mm）
- 材质（普通钢/高强钢/不锈钢/铝合金/自定义）
- 目标角度（0~180°）
- 弯曲半径（可选）
- 温度（可选，用于热弯工艺）

**推理引擎**：
1. **主引擎**：物理信息神经网络（PINN）
   - 融合材料力学先验知识
   - 基于历史加工数据持续在线学习
   - 支持每台设备的个性化模型微调
2. **Fallback 引擎**：规则引擎
   - 当数据不足或模型置信度低时自动降级
   - 基于材质、壁厚、管径的经验公式

**输出结果**：
- 推荐角度
- 补偿值
- 置信度评分
- 补偿说明（影响因素的分解）
- 相似历史案例参考

**数据闭环**：
```
加工完成 → 采集实际角度 → 计算残差 → 
上传云端 → 模型增量训练 → 参数表更新
```

### 4.2 多设备监控管理模块

**功能描述**：为管理者提供跨区域、多机台的实时状态监控和差异化控制能力。

**核心功能**：
- **设备总览看板**：地图/列表双视图，显示所有设备状态
- **实时状态监控**：
  - 在线/离线状态（心跳检测）
  - 任务执行状态（空闲/运行中/已完成/异常）
  - 当前设定角度 vs 实际角度
  - 加工偏差实时曲线
- **设备档案管理**：
  - 独立设备档案（型号、序列号、购入日期、维护记录）
  - 分组标签管理（按车间/产线/项目分组）
  - 差异化参数控制（每台机器可有独立的补偿参数表）
- **远程控制**：
  - 远程启停设备
  - 远程重置
  - 远程参数下发

**通信架构**：
- **实时通道**：WebSocket（浏览器 ↔ 服务器）+ MQTT（服务器 ↔ 设备）
- **心跳机制**：设备每 5 秒上报心跳，30 秒未收到标记为离线
- **断线重连**：自动检测断线并尝试重连，重连后同步最新状态

### 4.3 实时数据闭环处理模块

**功能描述**：实现"加工 → 采集 → 清洗 → 上传 → 优化"的完整数据闭环。

**数据流**：
```
PLC 传感器 → 边缘网关 → MQTT Broker → 后端服务
                                    ↓
                              数据清洗/校验
                                    ↓
                              残差计算（目标 vs 实际）
                                    ↓
                              模型增量训练（异步）
                                    ↓
                              生成新参数表
                                    ↓
                              版本管理 + OTA 下发
```

**边缘计算节点**（可选部署）：
- 本地缓存最近 1000 条记录
- 网络中断时本地推理（轻量模型）
- 网络恢复后批量同步

### 4.4 OTA 远程运维与版本管理模块

**功能描述**：实现补偿参数表、固件、软件的远程下发、版本管理和控制能力。

**核心功能**：
- **参数表管理**：
  - 多版本参数表并存
  - A/B 测试（灰度发布）
  - 版本回滚
- **下发控制**：
  - 单台下发 / 批量下发 / 分组下发
  - 定时下发
  - 下发状态追踪（已发送/已接收/已应用）
- **固件升级**（对接设备厂商）：
  - PLC 程序远程更新
  - 升级进度监控
  - 升级失败自动回滚

### 4.5 历史追溯与可视化分析模块

**功能描述**：提供完整的加工质量追溯、统计报表与数据可视化。

**核心功能**：
- **历史查询**：
  - 按时间范围、机器、操作员、材质等多维度筛选
  - 单条记录的完整追溯（输入参数 → 推荐参数 → 执行结果）
- **统计报表**：
  - 日/周/月产量统计
  - 合格率趋势
  - 偏差分布直方图
  - 设备稼动率分析
- **趋势分析**：
  - 补偿值随时间的变化趋势
  - 设备性能衰减预警
- **报告导出**：
  - PDF 报告（含图表）
  - Excel/CSV 数据导出

### 4.6 权限管理与安全审计模块

**功能描述**：确保不同角色只能访问特定权限范围内的数据和功能，记录所有关键操作日志。

**核心功能**：
- **RBAC 权限模型**：基于角色的访问控制
- **操作日志审计**：
  - 登录/登出记录
  - 参数修改记录
  - 任务提交记录
  - 用户/设备增删改记录
  - 数据导出记录
- **日志查询**：按用户、操作类型、时间范围筛选
- **异常告警**：多次登录失败、异常操作模式检测

### 4.7 告警与通知模块

**功能描述**：实时检测设备异常并推送通知。

**告警类型**：
- 设备离线告警
- 加工偏差超限告警
- 连续不合格告警
- 设备维护到期提醒

**通知渠道**：
- 站内消息
- 邮件通知
- 企业微信/钉钉 webhook
- SMS 短信（关键告警）

### 4.8 离线可用与边缘缓存

**功能描述**：确保在网络不稳定时依然能保障关键生产需求。

**核心功能**：
- **PWA 支持**：可添加到桌面，支持离线缓存静态资源
- **本地数据缓存**：最近 100 条历史记录本地存储
- **离线任务队列**：网络恢复后自动同步
- **本地推理 Fallback**：边缘节点部署轻量模型，断网时本地推理

## 五、数据库设计

### 5.1 实体关系图（ER）

```
users ||--o{ operation_logs : generates
users ||--o{ history_records : operates
machines ||--o{ history_records : produces
machines ||--o{ operation_logs : target
machines ||--o{ parameter_tables : has_versions
```

### 5.2 核心表结构

#### users（用户表）
| 字段 | 类型 | 说明 |
|---|---|---|
| id | UUID | 主键 |
| username | VARCHAR(50) | 用户名（唯一） |
| password_hash | VARCHAR(64) | SHA256 密码哈希 |
| role | VARCHAR(20) | 角色 |
| name | VARCHAR(50) | 显示名称 |
| created_at | DATETIME | 创建时间 |

#### machines（机器设备表）
| 字段 | 类型 | 说明 |
|---|---|---|
| id | VARCHAR(36) | 主键 |
| name | VARCHAR(100) | 机器名称 |
| location | VARCHAR(200) | 位置 |
| status | VARCHAR(20) | 在线/离线/维护中 |
| model | VARCHAR(50) | 设备型号 |
| serial_number | VARCHAR(50) | 序列号 |
| parameter_version | VARCHAR(20) | 当前参数表版本 |
| created_at | DATETIME | 创建时间 |
| last_active | DATETIME | 最后活跃时间 |

#### history_records（加工历史记录表）
| 字段 | 类型 | 说明 |
|---|---|---|
| id | VARCHAR(36) | 主键 |
| machine_id | FK | 关联机器 |
| task_id | VARCHAR(36) | 任务ID |
| diameter | FLOAT | 管径 |
| thickness | FLOAT | 壁厚 |
| material | VARCHAR(50) | 材质 |
| target_angle | FLOAT | 目标角度 |
| recommended_angle | FLOAT | 推荐角度 |
| recommended_offset | FLOAT | 补偿值 |
| actual_angle | FLOAT | 实际角度 |
| deviation | FLOAT | 偏差 |
| operator_id | FK | 操作员 |
| created_at | DATETIME | 创建时间 |
| completed_at | DATETIME | 完成时间 |

#### operation_logs（操作审计日志表）
| 字段 | 类型 | 说明 |
|---|---|---|
| id | INT | 自增主键 |
| timestamp | DATETIME | 操作时间 |
| username | VARCHAR(50) | 操作用户 |
| role | VARCHAR(20) | 用户角色 |
| action | VARCHAR(50) | 操作类型 |
| target_type | VARCHAR(50) | 对象类型 |
| target_id | VARCHAR(50) | 对象ID |
| detail | VARCHAR(500) | 详细描述 |
| ip_address | VARCHAR(50) | IP地址 |

#### parameter_tables（补偿参数表）
| 字段 | 类型 | 说明 |
|---|---|---|
| id | INT | 自增主键 |
| version | VARCHAR(20) | 版本号 |
| machine_id | FK | 适用机器（NULL 表示通用） |
| parameters | JSON | 参数表内容 |
| model_version | VARCHAR(20) | 生成该表的模型版本 |
| created_at | DATETIME | 创建时间 |
| is_active | BOOLEAN | 是否当前生效 |

## 六、API 接口设计

### 6.1 响应格式

所有接口统一返回格式：
```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 6.2 核心接口列表

| 接口 | 方法 | 说明 |
|---|---|---|
| `/api/auth/login` | POST | 用户/机器登录 |
| `/api/auth/logout` | POST | 登出 |
| `/api/auth/me` | GET | 获取当前用户信息 |

## 七、部署方案

### 7.1 开发环境
```bash
# 前端
cd frontend && npm install && npm run dev

# 后端
cd backend && python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
python main.py
```

### 7.2 生产环境（Docker Compose）
```yaml
services:
  db:
    image: postgres:15
    volumes:
      - pgdata:/var/lib/postgresql/data
  backend:
    build: ./backend
    environment:
      DATABASE_URL: postgresql://user:pass@db/dbname
  frontend:
    build: ./frontend
    ports:
      - "80:80"
  redis:
    image: redis:7
  mqtt:
    image: eclipse-mosquitto:2
```

### 7.3 边缘节点部署
- 轻量级 Docker 镜像
- 本地 SQLite + Redis
- 轻量 ONNX 模型（< 50MB）
- 支持 ARM64（树莓派/工控机）
